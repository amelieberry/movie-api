Movie API Documentation

Objective

The web application provides users with access to information about different movies, directors, and genres. Users are able to sign up, update their personal information, and create a list of their favorite movies.

Features

  • Return a list of all movies
  • Return (description, genre, director, image URL, whether it is featured or not) about a single movie by title to the user
  • Return data about a genre (description) by name/title (e.g., “Thriller”)
  • Return data about a director (bio, birth year, death year) by name
  • Allow new users to register
  • Allow users to update their user info (username, password, email, date of birth)
  • Allow users to add a movie to their list of favorites
  • Allow users to remove a movie from their list of favorites
  • Allow existing users to deregister

API Endpoints

Business Logic URL HTTP Method Parameters Request body data format Response body data format
Get a list of all movies /movies GET - - A JSON object holding data about all the movies
Get data about a single movie by title /movies/:title GET - - A JSON object holding data about a single movie, structured like:
{ "Title": "Movie One", "Description": "That's a description", "Genre": { "Name": "Comedy", "Description": "Comedy is funny stuff" }, "Director": { "Name": "Dir Ector", "Bio": "About Dir Ector", "Birth": 1988 }, "Actors": ["Act Or", "Ac Tor"], "ImageURL": "https://url.com", "Featured": false }
Get data about a genre /movies/genre/:genreName GET :genreName (the name of the genre (e.g. “Thriller”)) - A JSON object holding the name and description of a genre, structured like:
{ "Name": "Comedy", "Description": "Comedy is funny stuff" }
Get data about a director by name /movies/directors/:directorName GET :directorName (the name of the movie director) - A JSON object holding data about the specified director, structured like:
{ "Name": "Dir Ector", "Bio": "About Dir Ector", "Birth": 1988 }
Allow new users to register /users POST - A JSON object holding data about the user to add, structured like:
{ username: "Ano Nymous", password: "greatPassword3", email: "email3@mail.com", dataOfBirth: "3/1/1995", favoriteMovies: [] } 		A JSON object holding data about the new user, including their created ID, structured like:
{ id: 3, username: "Ano Nymous", password: "greatPassword3", email: "email3@mail.com", dataOfBirth: "3/1/1995", favoriteMovies: [] }
Allow users to update their user info /users/:username PUT :username (the username of the user) A JSON object holding data about the existing user, structured like: { username: "Ano Nymous", password: "greatPassword3", email: "email3@mail.com", dataOfBirth: "3/1/1995", favoriteMovies: [] } A JSON object holding the updated user data, including their ID, structured like:
{ id: 3, username: "Axel Doe", password: "greatPassword3", email: "email3@mail.com", dataOfBirth: "3/1/1995", favoriteMovies: [] }
Allow users to add a movie to their list of favorites /users/:username/movies/:movieID POST :username (the username of the user), :movieID (the Id of the added movie) - A JSON object holding the updated user data, including the new favorited movie, structured like:
{ id: 1, username: "Jane Doe", password: "greatPassword", email: "email@mail.com", dataOfBirth: "1/1/1995", favoriteMovies: ["Movie Two"] }
Allow users to remove a movie from their list of favorites /users/:username/movies/:movieID DELETE :username (the username of the user), :movieID (the Id of the added movie) - A text message indicating whether the movie was successfully removed from favorites
Allow existing users to deregister /users/:username DELETE :username (the username of the user) - A text message indicating whether the user was successfully deregistered
